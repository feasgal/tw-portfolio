# How-to: Annotate TDU source documents

Use this procedure when you need to mark up source files in TDU so you can see which parts of a source support specific documents or topics.

!!! goal

    Add annotations to source documents that are linked to entries in the document plan.

## Before you start

Confirm that you have:

-   Configured the **Central Data Path** and built the source list.
-   Imported or created the document plan.
-   Any needed correlations between documents and sources, if your team uses them.

## Steps

1.  Open the project and go to the **Sources** tab.
1.  Locate the source file you want to annotate.
1.  Double click the source entry, or select it and click **Open for annotation**.
1.  In the source viewer, navigate to the section you want to mark.
1.  Select the relevant text or area.
1.  Click **Add annotation**.
1.  In the annotation dialog:

    -   Choose the related document from the **Document** drop down list.
    -   Optionally, choose an annotation type such as **Requirement**, **Procedure**, or **Background**.
    -   Enter a short note that explains what this section covers.
1.  Click **Save annotation**.
1.  Repeat steps 5 to 8 for each additional section you need to annotate.
1.  When you finish, click **Save** or close the viewer if it saves automatically.

!!! outcome

    The source file now contains saved annotations, each linked to specific documents or topics in the document plan.
