# Get started with Acme licensing

This page is a fast path for new users. It links to the right topics in the documentation to help you install **Acme License Engine** and **Acme License Hub**, sign in, and run licensed apps.

Starting paths are below for two roles:

-   **End users**: People who need to run licensed apps
-   **Admins**: IT professionals who need to configure a local license server so that end users can run licensed apps using shared licenses

</div>

## Before you begin

!!! checklist "You need"

    -   Administrator rights on the machine
    -   Internet access
    -   An Acme account
    -   Optional for administrators: one product with sharable seats

## Get started

1.   Check [system requirements](/sample/reference/system-requirements.md){ target="_blank" }

    Review supported OS versions and network expectations. If your setup is out of scope, fix that first.

1.   [Install Acme License Engine and Hub](/sample/tutorials/install-hub.md){ target="_blank" } (tutorial)

    When the installer finishes, open **Acme License Hub** and sign in to confirm access.

1.   First success

    Pick a path based on your role.

    <div class="grid cards" markdown>

    -   **End users**

        ---

        1.  Use the Hub to [activate a license](/sample/how-to/licenses/activate.md){ target="_blank" } for your app.
        1.  Launch a protected app and confirm it starts without a license error.


    -   **Admins**

        ---
        
        1.  [Set up a basic local server](/sample/tutorials/setup-first-server.md){ target="_blank" } (tutorial).
        1.  [Connect one client](/sample/tutorials/connect-private-client.md){ target="_blank" } on the same LAN (tutorial).
        1.  On the client machine, launch a protected app and confirm it starts without a license error.



    </div>

1.   Next steps
